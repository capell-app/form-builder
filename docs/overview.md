# Form Builder

<!-- prettier-ignore-start -->

## What it does for you

Form Builder lets you define forms, render an active form on a compatible frontend surface, and collect what people submit. You add and reorder schema fields, set validation, configure storage and notifications, review submission payloads, and reply when a submitted email address is available.

## Your screens

- **Forms**: your form definitions, handle, site, active state, and submission count.
- **The form editor**: where you add, reorder, and configure schema fields.
- **Submissions**: everything people have submitted.

## What you can do

- Build a form by adding fields.
- Make fields required and validate them.
- Render an active form with the package's frontend form element and its handle.
- View a submission payload, then mark it read, spam, or archived.
- Configure notification and autoresponder settings for a form.

## Where to find it

Open **Marketing Studio > Forms** or **Marketing Studio > Form submissions** in the admin.

## Good to know

- Reorder schema fields and mark the ones people must fill in.
- A frontend form element resolves only an **active** form for the current site.
- Set a **Notification email** when staff should receive an email for each submission.
